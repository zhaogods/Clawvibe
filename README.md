# 让新手也看懂报错

基于 OpenClaw 的小项目：用户输入仓库地址、报错日志和触发步骤后，系统自动读取仓库、理解上下文、组织复现，并输出修复建议或补丁草案。

这个项目从第一版开始就接真实数据流：

- 真实仓库
- 真实报错日志
- 真实命令执行
- 真实运行结果

## 这个项目解决什么问题

新手处理陌生开源仓库时，通常不是不会看代码，而是不知道从哪里开始排查：

- 报错看不懂
- 仓库结构看不懂
- 不知道先跑什么命令
- 不知道该改哪个文件

这个项目只做一件事：

> 把“看见报错”到“知道怎么修”之间的那段路缩短。

## 输入

- 仓库地址
- 报错日志
- 触发步骤

## 输出

- 报错解释
- 项目结构摘要
- 相关文件定位
- 真实复现记录
- 根因判断
- 修复建议
- 补丁草案
- 验证步骤

## 半天内完成的版本

半天内只做最核心的真实闭环：

1. 接收输入
2. 拉取并读取真实仓库
3. 识别项目类型
4. 解释报错
5. 执行真实命令并拿到结果
6. 生成修复建议报告
