# 关键发现

## 比赛相关

- 比赛是阿里云体系下的 OpenClaw 挑战赛。
- 项目核心应是一个可运行的小应用，不是先写论坛稿。
- OpenClaw 适合本项目，因为它本身擅长文件读取、终端执行和工作流编排。

## 主题相关

- 当前目录最适合的参赛主题已经固定为：
  `让新手也看懂报错：我用 OpenClaw 做了个会读仓库、会复现、会给补丁的 vibe coding 助手`
- `初稿.md` 是最接近项目本体的素材。
- `比赛论坛投稿初稿.md` 更适合提供“先判断、后执行、结构化输出”的方法论借鉴。

## 落地相关

- 半天内应该优先完成：
  - 输入表单或命令行入口
  - 仓库扫描
  - 报错解释
  - 相关文件定位
  - 真实复现执行
- 当前真实数据流原则：
  - 不用 mock
  - 不用假数据
  - 不把建议执行写成已执行
  - 第一版就必须读取真实仓库并执行真实命令
